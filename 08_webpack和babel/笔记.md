## 为什么需要babel
- 使用es6语法，使用ts,使用jsx，这些都需要babel来转移
- 核心是 **babel-core** 来处理转移
- 转移的target要根据 **babel-preset/env**
- **babel-loader**则是连接webpack和babel
- 容易搞不清楚core-js和babel的区别 
- babel是转译语法的，比如const 箭头函数
- core-js是转译es6没有的方法的，比如promise,array.includes打polyfill
- 那为什么vite不需要babel或者很少需要呢？因为都是esbuild来做的 ,但是依然需要core-js

## babel的底层原理
      parsing        transformation         code generation
- es6  --->    原ast     --->         新ast      ---->         es5
- transformation的过程遍历树进行操作，可以应用不同插件(预设里面就是这些插件)，进行不同的transform
## 使用
- 可以手动使用plugin
- preset/env，除此之外，还需要其他的，可以在后面加一个{targets:''},也可以用browserslistrc,targets优先级更高，browserslistrc也会在package.json里面
- 项目中可能除了js之外比如jsx都需要babel，所以要抽离到特定的文件中， **babel.config.json**更推荐（可以作用于monorepo的子包）  **.babelrc.json**


## 认识polyfill
- 像promise,array.includes，async，await等等他们都是不支持的，需要加入polyfill
- polyfill引入一个core-js即可，但是core-js不支持async
- 两种办法一种加入插件@babel/plugin-transform-runtime，
- 一种加入regenerator-runtime(usebuiltIns) 添加到全局，可能会污染别人的代码
- 需要排除exclude node_modules

react的话使用 特定的preset/react
