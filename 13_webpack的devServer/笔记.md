## webpack --watch
- 对所有的文件重新进行编译
- 编译成功后，生成新的文件
- 每次重新刷新整个页面

## webpack-dev-server
- webpack serve
- 也是重新编译文件，写入到内存中（如果启用HMR才不是全量更新）
- webpack dev server使用express

## webpack-dev-middleware
- 很少使用
- 可以开启一个服务来把webpack打包后的文件跑到某个端口

## HMR (hot module replace)
- 开启 hot:true，但是仍然是全量刷新，需要指定module
```
if (module.hot) {
  module.hot.accept("./math.js", function () {
    console.log("math.js更新了");
  });
}
```

## 框架启用 HMR
- 社区都有很成熟的库了，比如vue-loader实现vue的HMR,react有react-refresh


## HMR原理
- webpack-dev-server会创建两个服务一个express提供静态资源，一个socket服务
- socket双方可以双向通信，服务器可以发送文件给客户端
- 当服务器监听到对应模块变化的时候，生成两个文件，一个manifest和js(update chunk)
- 通过长连接发送给浏览器
- 浏览器拿到两个文件后，通过HMR runtime加载这两个文件，并且针对的更新

## devServer里面的其他属性
- static  静态资源从哪里来
- publicPath  打包结果怎么被访问
webpack打包出来的资源
- 分为源码（js/css/懒加载的chunk）
- 静态资源（HTML/图片/public目录）
### output.publicPath
```
output: {
  filename: "bundle.js",
  publicPath: "/assets/"
}
浏览器访问就要这样
<script src="/assets/bundle.js"></script>
CDN场景
output: {
  publicPath: "https://cdn.xxx.com/"
}
```
### devServer.static.directory
```
devServer: {
  static: {
    directory: path.join(__dirname, "public")
  }
}
http://localhost:8080/index.html
http://localhost:8080/logo.png
访问真实存在在public目录里面的资源
```
### devServer.devMiddleware.publicPath
```
output: {
  filename: "bundle.js"
},
devServer: {
  devMiddleware: {
    publicPath: "/dist/"
  }
}
浏览器访问
/dist/bundle.js
```

配置	解决的问题
output.publicPath	👉 webpack 产物“从哪访问”
devServer.static	👉 静态文件“从哪来”(提供了一个文件服务器)，可以直接http://localhost:8080/favicon.ico
devMiddleware.publicPath	👉 内存文件“挂在哪”
