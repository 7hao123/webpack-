loader用于对模块的源代码进行转换
在加载这个模块的时候，webpack不知道如何进行加载的，我们必须指定对应的loader

1.内联方式
import 'css-loader!../css/index.css';加载这个css的时候使用css-loader
2.cli方式（已经删除了）
3.配置方式

css-loader   负责.css的解析，不会将解析后的css插入到页面中
style-loader 插入style  在header里面生成style
postcss-loader 通过JavaScript来转换样式的工具，进行css的转换和适配，自动添加浏览器前缀（autoprefixer）,(postcss-preset-env)给css加上polyfill和加上前缀
css和less都需要配置postcss-loader，直接在根目录


如果在css中 @import其他css的话 postcss不会执行，所以要给css-loader配置option


browserslist   是一个在不同前端工具（babel,autoprefixer,postcss-preset-env）共享目标浏览器和nodejs版本的配置
 caniuse usage-table  占有率
条件查询使用 caniuse-lite去查询匹配到哪些浏览器
